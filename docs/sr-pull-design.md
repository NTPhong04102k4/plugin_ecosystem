# Thiết kế: `sr pull` — cầu nối deterministic authswagger → data layer + digest

> Trạng thái: **ĐÃ CHỐT (frozen)** — chờ duyệt để code. Không code trước khi duyệt.
> Ngày chốt: 2026-07-20.

## 1. Mục tiêu & vấn đề

Mối nối **skillrunner ↔ authswagger** (spec → code data-layer) gần như **100% máy móc
(deterministic)** — không cần suy luận. Nếu để Claude ngồi giữa seam đó thì đốt token vô ích ở
hai chỗ đắt nhất:

1. Cho Claude **đọc OpenAPI thô** (spec HR/QLPK ~50–200k token).
2. Cho Claude **tự viết types** (output token lớn theo schema).

`sr pull` biến cả seam này thành **một lệnh 0 token**: fetch spec → lọc tag → codegen types →
instantiate template data-layer → in ra **digest gọn** (< 2k token). Claude chỉ đọc digest và
chỉ tiêu token ở phần thật sự cần não: **build UI + nối UI ⇄ service**.

## 2. Vị trí trong kiến trúc

Kiến trúc đã chốt (xem cuộc trao đổi): **Claude là orchestrator, không dùng n8n.**

```
authswagger ──fetch──► lọc tag ──► codegen types.ts ──► instantiate ──► emit DIGEST gọn
  (spec thô)          (stdlib json) (openapi-typescript) template pack        │
        └──────────────── LÀN 0-TOKEN (sr pull, không có Claude) ────────────┘
                                                                             ▼ (~0.5–2k token)
                                              CLAUDE: đọc digest → build UI + nối ⇄ hook
                                              (KHÔNG BAO GIỜ đọc spec thô)
```

`sr pull` là làn 0-token. Claude vào sau, chỉ ăn digest.

## 3. Quyết định đã khóa

| # | Quyết định | Chốt |
|---|---|---|
| 1 | Preflight authswagger | **KHÔNG tự run.** `sr pull` chỉ health-check; nếu chết → in lệnh cho user tự chạy rồi thoát non-zero |
| 2 | Codegen types | **openapi-typescript** (TS, chạy qua `npx`). Cần Node trên máy |
| 3 | Vị trí cache | **`docs/context/` trong repo đích** (theo git, review được) |
| 4 | Service layer | **react-query hook** (theo `qlpk/docs/react-query-usage.md`), không phải fetch trần |

## 4. Command & flags

```
sr pull --tag <tag> [flags]
```

| Flag | Ý nghĩa | Default |
|---|---|---|
| `--from <url>` | URL spec authswagger | `http://localhost:8080/openapi.json` |
| `--env <env>` | môi trường authswagger | `dev` |
| `--spec <n>` | nhóm spec (dropdown) | — (dùng `--tag`; hoặc kết hợp) |
| `--tag <tag>` | lọc endpoint theo tag (vd `Khoa`) | **bắt buộc** |
| `--out <dir>` | nơi ghi file sinh ra | `src/api/<tag>` |
| `--base <proxyBase>` | baseUrl proxy | tự đọc từ `servers[]` → `/env/<env>/api` |
| `--codegen ts` | bộ sinh types | `ts` (openapi-typescript) |
| `--dir <project>` | repo đích (detect stack/pack) | cwd |
| `--no-cache` | ép chạy lại, bỏ qua cache | off |

## 5. Pipeline (toàn bộ 0 token)

```
1. Preflight  : GET <host>/healthz.
                Chết → in: "authswagger chưa chạy. Chạy: cd <WorkFlowAutomation> && go run ."
                → exit code != 0. KHÔNG tự spawn.
2. Fetch      : GET spec (kèm Basic Auth header nếu authswagger bật BASIC_AUTH_*)
3. Parse      : stdlib encoding/json (metadata nông) → lọc operations theo --tag
                Tag không tồn tại → liệt kê các tag có sẵn rồi thoát
4. Resolve    : proxyBase từ servers[] (vd /env/dev/api); ghi đè bằng --base nếu có
5. Codegen    : filterSpecByTag → npx openapi-typescript → types.ts
                npx/Node thiếu → báo lỗi rõ (không fallback im lặng)
6. Scaffold   : instantiate template react-query của pack → <tag>.hooks.ts (gọi qua proxyBase)
7. Digest     : in JSON gọn ra stdout + ghi cache vào docs/context/pull-<tag>.json
```

## 6. Output

**File ghi vào `--out/`** (mỗi file có header `// generated by sr pull — do not edit`):
- `types.ts` — types từ openapi-typescript
- `<tag>.hooks.ts` — react-query hook cho từng endpoint (useQuery/useMutation), gọi qua proxyBase
- `index.ts` — re-export

**Digest** (stdout — thứ DUY NHẤT Claude đọc, mục tiêu < 2k token):

## 7. Cache

- **Key** = `hash(from + env + spec + tag + codegen + specContentHash)`.
- **Vị trí**: `<project>/docs/context/pull-<tag>.json` (theo git).
- **Hit** (specContentHash trùng) → in digest cache, bỏ qua fetch/parse/codegen.
- **Đổi spec** (hash khác) → tự regen. `--no-cache` → ép chạy lại.

## 8. Determinism & an toàn

- Sort keys, thứ tự endpoint/field ổn định → same spec = same output (đúng rule `technical`).
- Chỉ ghi trong `--out` + `docs/context/`; không đụng file khác.
- Không overwrite file thiếu header `generated by sr pull` (đúng rule "không overwrite file
  không do tool sinh").
- `sr pull` **không cần approval** (deterministic, chỉ data layer). Approval vẫn ở 2 gate do
  Claude giữ: **duyệt plan** + **duyệt commit**.

## 9. Quan hệ với skill cũ

- `sr pull` = phần data-layer + digest (0 token), thay cho việc Claude đọc spec trong
  `spec-from-source` / `scaffold-data`.
- Sau `sr pull`, Claude chỉ chạy `build-ui` + nối UI ⇄ hook + verify.
- `deliver-feature` gọi `sr pull` ở bước data thay vì để Claude ăn spec thô.

## 12. Phụ thuộc & ghi chú hiện thực (đã code — cập nhật 2026-07-20)

- `packs/react.json` có `codegen` trỏ tới `packs/assets/react/templates/{hooks,index}.ts.tmpl`
  (đã thêm ở bước A).
- Node/npx có sẵn (cho openapi-typescript).
- **KHÔNG dùng `kin-openapi`.** Giữ nguyên tính chất *zero-dependency* của binary: parse spec
  bằng stdlib `encoding/json` (chỉ cần metadata nông: operationId/method/path/params/body). Việc
  dựng type nặng đã có openapi-typescript lo. `go.mod` vẫn 0 dependency.
- **Lọc spec theo tag trước khi codegen** (`filterSpecByTag`): cho openapi-typescript ăn spec đã
  rút gọn còn operation của tag đó (giữ nguyên toàn bộ components). Lý do phát hiện khi test thật:
  spec QLPK **dùng lại operationId giữa các tag** (vd `GenerateCode`, `GetFeesByService`) → nếu cho
  ăn cả spec, `operations` bị trùng key → types.ts không compile. Lọc theo tag khử trùng + giảm
  kích thước (Khoa: types.ts từ ~8800 → ~600 dòng).
- **Robust operationId thiếu:** endpoint không có operationId → fallback type `unknown`
  (`HasOp=false`, không tham chiếu `operations`), không vỡ TS.
- Đã verify thật: `sr pull --env dev --spec 6 --tag Khoa` → 3 endpoint (GetKhoaList/SearchKhoa/
  SyncKhoa), `tsc --noEmit` exit 0, `OpRes` trích đúng type (không phải `unknown`).
